After uploading and applying a **Data Source** to your test case, the next crucial step is to **configure how the test case should execute** using your attached data.

Robonito allows you to easily define an **Execution Strategy** to control exactly how many times and in what specific order your automated test case should run through the uploaded data rows.

---

## Configuring an Execution Strategy

Follow these initial steps to open your execution settings:

### 1. Open the Test Explorer

- Navigate to the **Test Explorer** from the main Robonito dashboard.
- Select the specific test case that you want to properly configure.

![Test Explorer View](/data-source/test-explorer.png)

### 2. Open the Execution Drawer

- In the top menu bar, click the **Execute** button.
- A side drawer will slide open, listing all available browser configurations associated with your test (e.g., Chrome, Firefox, Edge).

![Open Test Case Config Drawer](/data-source/execution-config-drawer.png)

### 3. Edit a Browser Configuration

- Click on any of your listed browser configurations.
- A secondary drawer will open, allowing you to edit the detailed settings for that specific execution profile.

![Edit Execution Config](/data-source/edit-execution-config.png)

---

## Applying Data Source Strategies

Inside the detailed configuration drawer:

1. Click on the **"Apply Data Source"** option.
2. The **Data Source Strategy Configuration Panel** will become visible.
3. Choose your preferred execution strategy and fill in the required details.

![Apply Data Source Strategy Panel](/data-source/apply-data-source-config.png)

---

## Available Execution Strategies

Robonito supports two primary execution strategies for data-driven testing:

### 1. Sequential Strategy

The Sequential Strategy executes the test case iteratively in the **exact order** of the rows provided in the data source.

- **Use Case:** Best when you want to thoroughly test specific, predictable rows in chronological order.
- **Range Control:** You can specify a targeted **range** of rows to execute out of the entire dataset.

**Example Configuration:**
```yaml
strategy: sequential
range:
  start: 1
  end: 10
```
*Result: This specific strategy will run the automated test case exactly from row 1 to row 10 sequentially.*

![Sequential Strategy Setup](/data-source/apply-data-source-config.png)

---

### 2. Random Strategy

The Random Strategy executes the test case iteratively by fetching completely random rows from the associated data source.

- **Use Case:** Best when you want to simulate chaotic, randomized user behavior or stress test inputs unpredictably.
- **Iteration Control:** You simply specify exactly how many times the test case should execute globally.

**Example Configuration:**
```yaml
strategy: random
times: 5
```
*Result: This specific strategy will execute the test case exactly 5 times, picking a completely random row from your dataset for each run.*

![Random Strategy Setup](/data-source/sequential-strategy.png)

---

## Final Step: Saving the Configuration

Once you've configured your desired data strategy:

1. Click the **Save** or **Update** button located at the bottom of the configuration drawer.
2. The next time you execute the test case, Robonito will automatically run it using your newly selected iteration strategy and data source values.
