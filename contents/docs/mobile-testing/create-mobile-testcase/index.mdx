# Creating a New Mobile Test

This guide explains exactly how to seamlessly create a brand new mobile test case using the **Robonito** testing dashboard.

---

## Adding Test Cases to a Suite

In Robonito, test cases are systematically organized within **Test Suites** based on their underlying functionality or application modules. To create mobile-specific tests, you must ensure you are actively working within a suite designed explicitly for mobile testing.

### Steps to Create a Mobile Test Case

Follow the essential steps below to properly configure and create a new mobile test case:

1. **Select the Target Suite**  
   Choose an existing test suite from the **Target Suite** dropdown menu. Alternatively, you can type a brand new name to dynamically create a fresh suite where this new test case will reside.  
   ![Target Suite Selection](/test-case/target-suite.png)

2. **Enter the Test Case Name**  
   Provide a clear and descriptive title in the **Test case name** field to help you and your team easily identify this specific test scenario later.  
   ![Test Case Name Field](/test-case/test-case-name.png)

3. **Select the Test Type**  
   Select the **Mobile** option as your primary test case type. This strictly defines the test as a mobile automation script and unlocks device-specific testing tools.  
   ![Mobile Test Type Selection](/test-case/mobile.png)

4. **Add a Description (Optional)**  
   Enter a concise description explaining the overall purpose and expected scope of the test case. Providing detailed context here is highly recommended for collaborative teams.

5. **Select Your Platform**  
   Because mobile ecosystems inherently differ, you must explicitly choose the target mobile platform (e.g., iOS or Android) on which this automated test will be launched and recorded.  
   ![Platform Selection Configuration](/test-case/platfrom.png)

6. **Create the Test Case**  
   Click the **Create** button to finalize and save the test configuration. Once successfully created, your new mobile test case becomes instantly available for recording actions and active script execution.
