# API Testing

Robonito provides a powerful and intuitive interface for creating, testing, validating, and automating API requests. With built-in support for request configuration, variable extraction, and response assertions, you can easily validate backend services without writing a single line of code.

This guide covers all the major features of API testing in Robonito.

### What You Will Learn

Explore the topics below to get started with building and managing robust API tests:

- **[Create an API Test Case](./create-api-testcase)**  
  Learn how to set up your first API test case within a test suite.

- **[Configure API Request Details](./configure-api-endpoint)**  
  Master endpoint configurations, including query parameters, HTTP headers, and authentication methods.

- **[Use Variables in Your Tests](./variables-in-api)**  
  Discover how to extract dynamic values from responses to seamlessly chain multiple API requests together.

- **[Add Assertions to Validate Responses](./assertions)**  
  Ensure your APIs return the correct HTTP status codes, performance times, and JSON data payloads.

- **[Understand Execution Reports](./execution-report)**  
  Dive into debugging and viewing detailed execution histories, including success and failure logs of your tests.
