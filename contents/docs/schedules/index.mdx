# Test Case Scheduling

Robonito intimately allows you to intelligently schedule test cases to smoothly run automatically at precisely specified times. Scheduling helps automation teams confidently run tests on a regular cadence without manual intervention, making it incredibly ideal for robust regression testing, nightly parallel runs, and continuous validation pipelines.

---

## Creating a New Schedule

To completely automate your suites, simply navigate to the **Schedules** section in the sidebar and click the **New Schedule** action button. The dedicated schedule dialog will open, where you can easily configure exactly when and how your test cases should run blindly.

![Schedules Dashboard Overview](/test-case/schedule.png)

### Schedule Configuration Fields

The configuration wizard natively guides you through setting up your automated timeline:

![New Schedule Configuration](/test-case/new-schedule.png)

- **Schedule Name:** Enter a highly unique and descriptive name for the automated schedule (e.g., *Daily Regression*, *Nightly API Tests*). Maximum length is safely capped at 50 characters.
- **Frequency:** Select exactly how often the selected test cases should structurally run. (e.g., **Daily** executes the designated test cases every single day at the explicitly specified time).
- **Timezone:** Natively select the geographical timezone in which the schedule should anchor and run (e.g., `Asia/Calcutta`). Live execution time is automatically calculated based strictly on the selected timezone.
- **Execution Time:** Select the exact 24-hour time when the formally scheduled execution should start (e.g., `11:00 PM`).
- **Environment (*):** Select the target infrastructure in which the automatically scheduled test cases will reliably execute (e.g., `LOCAL` or `CLOUD`). *This field is mandatory.*

---

## Selecting Test Cases

Directly below the configuration fields, you must explicitly select the exact test cases to include in your new automated timeline.

**Test Case Selection Features:**
- Seamlessly search test cases by literal name, description, or custom tag.
- Dynamically select individual test cases using checking boxes securely.
- Test cases are natively grouped logically for remarkably easy navigation.
- Each test case visibly displays its dedicated type indicator (e.g., `WEB`, `API`, `MOBILE`).
- *Note: A new schedule fundamentally cannot be created until at least one test case is actively selected safely.*

### Committing the Schedule
After fully configuring all field parameters and selecting the target test cases:
1. Carefully review all the populated schedule details natively.
2. Click the primary **Create Schedule** action button securely.

**Once successfully created:**
- The new schedule becomes logically active immediately.
- The securely selected test cases will predictably run fully automatically at the formally next scheduled time naturally.

## Managing Your Schedules

All safely created structural schedules are permanently displayed in the native **Schedules** dashboard table locally.

![Schedules Table Dashboard](/test-case/schedule-report.png)

### Schedule Table Columns

| Column Name | Description |
| :--- | :--- |
| **Name** | Visually displays your custom descriptive schedule name natively. |
| **Status** | Indicates whether the timeline is currently active. (e.g., `ACTIVE` – Schedule is formally enabled and will faithfully run automatically). |
| **Upcoming Run** | Intelligently shows precisely when the very next scheduled execution will occur natively (e.g., `in 10 hours`). |
| **Frequency** | Displays the recurring execution frequency, specific time, and strict timezone naturally (e.g., `Daily 23:00 hours : Asia/Calcutta`). |
| **Scheduled By** | Visually displays the authenticated user name who manually created the custom timeline. |
| **Created** | Responsively displays exactly when the recurring schedule was originally authored. |
| **Actions** | Native row actions elegantly allow you to **Pause** (temporarily disable), **Edit** (modify configuration variables), or securely **Delete** (remove permanently) the given active schedule seamlessly. |
