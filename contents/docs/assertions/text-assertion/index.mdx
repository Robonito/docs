# Text Assertions

Robonito supports powerful content-based assertions to help you validate the text and messaging displayed on any given page.

When adding an assertion to your test case, you can select from various categories of assertions to match your exact validation strategy:

## Content-Based Assertions

Content-based assertions scan and validate the visible UI text displayed on the screen. Robonito provides three primary types of text-matching validation rules:

### 1. Plain Text
Asserts that a specific, static string of text appears on the page.

- **Matching Support:** Supports both **Exact** or **Partial** text matching, depending on your strictness requirements.

**Use Case:**  
Validate stable, static content such as page headings, button labels, and standard confirmation messages.

---

### 2. Custom Expression
Asserts text matching against a dynamic custom expression, designed for more advanced testing scenarios.

**Use Case:**  
Validate dynamic runtime text such as generated User IDs, live timestamps, order numbers, or stored API variables.

---

### 3. Regex (Regular Expressions)
Validates UI text by evaluating the element against a robust Regular Expression.

**Use Case:**  
Match complex or unpredictable text formatting patterns, such as validating email addresses, postal codes, or dynamically generated alphanumeric strings.
