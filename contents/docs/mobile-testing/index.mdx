# Mobile Testing with Robonito

Robonito provides a powerful architecture to test mobile applications natively, ensuring they function correctly across different operating systems and virtual devices. This section guides you through the tools and configurations designed for recording and analyzing mobile test cases.

---

## Supported Platforms

Robonito's automation engine supports end-to-end mobile testing for the following ecosystems:
- 🤖 **Android Applications** (via `.apk` uploads)
- 🍏 **iOS Applications** (via `.ipa` uploads)
- 💻 **Emulator-Based Execution** (Android Studio / Appium engines)

---

Navigate through the core topics below to configure your local setup and begin recording robust mobile tests:

- 📱 **[Create a Mobile Test Case](./mobile-testing/create-mobile-testcase)**  
  Learn how to initialize a new mobile test script within your workspace suite.

- ⚙️ **[Configure Mobile Setup](./mobile-testing/mobile-setup)**  
  Install necessary global dependencies (Appium, SDK tooling, Node.js) required to operate a mobile emulator on your machine.
  
- 🎥 **[Start Recording a Mobile Test](./mobile-testing/recording)**  
  Learn how to upload your `.apk` or `.ipa` app files, inherently boot the emulator, and seamlessly record your touch interactions.

- 🅰️ **[Variables in Mobile Tests](./variables)**  
  Pass dynamic dataset parameters directly into your mobile workflows using standard variable management.

- ✅ **[Add Mobile Assertions](./assertions)**  
  Validate specific native mobile application UI element states automatically during test execution.

- 📊 **[Mobile Execution Reports](./reports)**  
  Analyze success metrics, trace execution forensics, and review detailed timeline executions in the dashboard.
