# Applying a Data Source to a Test Case

Once you have successfully uploaded a **Data Source** (like a `.csv` or Excel file) to your workspace, the next step is to **connect it to your test cases**. 

This process enables powerful data-driven testing by dynamically populating your automation inputs with values natively fetched from your attached spreadsheets row by row.

---

## The Parameterization Workflow

To begin injecting data source values into your test flow, follow these steps:

### 1. Open the Test Explorer

1. Navigate to the **Test Cases** view from the main Robonito dashboard.
2. Select and open the specific test case you wish to parameterize.

![Test Explorer View](/data-source/test-explorer.png)

### 2. Select an Interactive Node

1. Click on any test step within the timeline that requires text to be entered (e.g., a **Fill Input** or **Type** node).
2. The step configuration panel will slide open on the right, allowing you to edit the **Input Value**.

![Edit Test Node Panel](/data-source/edit-node.png)

---

## Injecting Data Source Values

Within the **Input Value** field, you can access your stored data sources seamlessly using Robonito's autocomplete syntax:

1. **Trigger the Quick Menu:** Start typing `{{@` precisely in the input field. This prefix automatically opens a dropdown menu displaying all your locally uploaded data sources.
2. **Select the File:** Click on the desired **Data Source File Name** from the dropdown list.
3. **Select the Column:** A secondary popup will immediately cascade to the right. Select your desired **Column Name** from that specific data sheet.
4. **Completion:** After selecting the column, the field expression string will automatically format using the correct engine syntax:

   ```text
   {{@FileName.ColumnName}}
   ```

### Working Example

If your uploaded file is officially named `users.csv` and it contains a column explicitly titled `email_address`, your final field expression would correctly look like this:

```text
{{@users.email_address}}
```

![Select Data Source Popup](/data-source/fill-data-source.png)  
![Select Data Source Column Popup](/data-source/fill-data-source-column.png)

This explicit `{{@...}}` syntax commands Robonito's runtime engine to dynamically fetch the exact value from the `email_address` column corresponding to the current loop iteration during an execution sequence.

---

## Important Injection Rules

Keep the following technical rules in mind when parameterizing fields:
- **The Correct Trigger:** The data source fetching popup **only** appears when you type the exact characters `{{@`. (Regular variables use `{{`, while random values use `{{$`).
- **Exact File Accuracy:** The file name and column name must reflect exact matches with the uploaded source schema.
- **Preview Capability:** You can visually preview your selected bindings directly within the test runner before officially kicking off the test loop.
