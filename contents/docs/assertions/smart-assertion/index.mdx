# Smart Assertions

Smart Assertions allow you to define UI validations using simple, human-readable descriptions. Instead of manually configuring complex assertion rules, you describe what should be true about a selected UI element, and **Robonito** automatically interprets and applies the appropriate validation logic.

The **Smart Assertion** dialog appears when you choose to add a smart validation during an active recording or while editing an existing test step.

![Smart Assertion Dialog](/test-case/smart-assertion1.png)

---

## Creating a Smart Assertion

The **Create Assertion** dialog enables you to define intelligent, flexible assertions using natural language.

### Assertion Description

Enter a clear and accurate description of what exactly should be true about your selected UI element.

**Examples:**
- *"The login button should be visible and clickable"*
- *"The error message should contain the text 'Invalid credentials'"*
- *"The submit button should be disabled"*

> **Note:** You can enter up to **500 characters** in the description box.

---

## Instructions for Writing Smart Assertions

To create highly effective Smart Assertions, follow these best practices:

1. **Select the Target:** First, click and select the target UI element on the page.
2. **Describe the Verification:** Describe exactly what you want to verify about the element.
3. **Use Specific Language:** Use clear and specific declarative language. Common functional keywords include:
   - `should be visible`
   - `should be clickable`
   - `should contain text`
   - `should be disabled`
4. **Save:** Save the assertion once your description is complete.

The assertion is then automatically evaluated by Robonito during test execution.

---

## Saving Your Assertion

To finalize your configuration:

1. Click the **Save Assertion** button to attach the smart assertion to your selected test step.
2. During test execution, Robonito evaluates this natural language assertion and dynamically marks it as either **Passed** or **Failed** in your test's execution report.

---

## Benefits of Smart Assertions

Using Smart Assertions dramatically improves your testing workflow by providing:
- **Faster Assertion Creation:** Skip manual configurations and complex property dropdowns.
- **Reduced Manual Effort:** No need to understand inner element properties, XPath, or DOM structures.
- **Clear, Readable Validations:** Anyone on your team can read and understand the assertion intent immediately.
- **Improved Test Maintainability:** Natural language tests are much more resilient to minor application code adjustments.
