# Execution Reports

The **Execution Reports** section in Robonito provides a comprehensive, centralized dashboard covering all of your automated test executions. It allows you to reliably monitor active testing activity, actively track failure occurrences across environments, analyze execution trends over time, and comfortably review deeply detailed test results.

These reports natively help QA teams maintain full visibility into their automation performance and quickly identify unstable or failing test suites.

## Execution Trend Graph

The dynamic trend chart flawlessly visualizes your test runs over hours, days, or weeks.

It displays execution histories intuitively categorized by:
- **GitHub Check Runs**
- **Scheduled Executions**
- **Entire Test Suites**
- **Single Test Cases**

Using the trend graph effectively helps enterprise teams:
- Identify standard native test execution frequencies dynamically.
- Detect sudden visual spikes in system-wide automation failures.

- Securely monitor the cumulative health of your automation architecture daily.

## Failed Tests Panel

On the right side of your primary dashboard, the **Failed Tests** panel actively highlights your most recently failed test cases natively.

Each individual failed entry reliably shows the following metadata:
- **Test Case Name:** The specific script or test that actively failed.
- **Suite Name:** The parent suite where the test natively lives.
- **Time of Failure:** The precise timestamp of the automated failure.
- **Failure Reason:** A clear, automated error description highlighting exactly why it failed.
- **Environment:** Where it executed securely (e.g., `LOCAL` or `CLOUD`).
- **Test Type:** The platform classification (e.g., `WEB`, `API`, `MOBILE`).
- **Tags:** Any custom organizational tags actively applied to the specific test.

**Investigating Failures:**
- Click **Show More** to securely expand the detailed failure breakdown metrics.
- Click **View Logs** to instantly inspect the deeply detailed execution terminal logs.

*This focused panel successfully allows for rapid pipeline debugging without having to navigate blindly through giant, full-length test reports.*

## Advanced Filtering

Robonito provides robust, native dashboard sorting filters to efficiently narrow and explicitly refine your report data for focused analysis:

- **Status:** Instantly filter target executions actively by `Passed`, `Failed`, or currently `In Progress`.
- **Environment:** View automation metrics exclusively mapped from `Local` or `Cloud` execution environments.
- **Execution Type:** Safely narrow massive reports down natively by a single `Test Case`, an entire `Suite`, or a scheduled automated `Schedule`.

### Downloading Reports
You can reliably click the **Download** action button to natively export execution logs or comprehensive `.csv` reports securely to save or share externally outside of the Robonito platform.
