# Test Case Execution

Robonito allows you to execute your automated test cases using highly configurable execution environments. The **Execution Configuration** dialog enables you to explicitly define where and how a test case should run before starting the automation engine.

This dialog appears immediately when you click the **Execute** button on any test case in your suite.

![Execution Configuration Dialog](/test-case/execution.png)

---

## Execution Configuration

The **Execution Configuration** panel provides granular options to control the specific test execution location and your underlying environment variables.

### 1. Runs On (\*)

Select the explicit infrastructure where you want to execute your test case.

- **[Local Execution](./executions/local):** Executes the test case on your current local machine using your natively installed Robonito setup.
  - *Ideal for:* Active test development, immediate debugging, and quick script validation.
- **[Cloud Execution](./executions/cloud) (Premium):** Executes the test case entirely on Robonito-managed scalable infrastructure.
  - *Ideal for:* Scalable remote automation, team-shared testing pipelines, and unattended CI/CD scheduled runs without relying on local system resources.

### 2. Environment (\*)

Select the target software and network environment in which the test case should execute exactly.

- **Local:** Executes the active test strictly using your locally configured settings, variables, and browser resources.

- The **Execute** button dynamically disables itself.

## Executing a Test Case

To start your automation run after selecting your desired execution configuration:

1. **Review:** Double-check your selected **Runs On** and **Environment** dropdown values.
2. **Execute:** Click the primary **Execute** button to initiate the run.

Once the automated execution officially starts:
- The test case begins running sequentially with the exact selected configuration.
- Your detailed execution progress is tracked and displayed in real-time.
- Robust visual and data results are generated immediately upon completion.

---

## Execution Behavior & Results

During active test execution, Robonito's automation engine will autonomously:
- Run every single programmed test step sequentially in explicit order.
- Dynamically mark specific steps as **Passed** or **Failed** dependent on assertion outputs.
- Capture deep execution logs, runtime variables, and status metrics.

After the execution fully completes:
- Every step clearly shows its final execution status marker.
- Failed steps explicitly highlight the exact reason or UI error behind the failure.
- Assertion results and expected comparisons are displayed clearly within the UI.
- All execution data is securely available in your Test Logs for in-depth review and debugging.

---

## Why Execution Configuration Matters

Properly setting your execution configuration natively ensures:
- All tests securely run in the correct, predictable network environment.
- Highly repeatable, stable automation results across all team members.
- Tightly controlled execution behaviors across parallel suites.
- Maximum flexibility when swapping between local script debugging and cloud-based test deployments.
