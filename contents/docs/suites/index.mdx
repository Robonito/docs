# Managing Test Suites

Suites are essential organizational components within Robonito. They function as centralized folders or containers designed to group related test cases together based on your application's architecture or specific feature modules. 

The suite management module features a powerful, IDE-like interface that allows your team to create, edit, organize, and execute entire batches of tests simultaneously.

---

## 1. Creating a New Suite

To logically isolate a new set of tests, you must first create a dedicated suite:

1. **Click "+ New Suite":** Click the `+ New Suite` button located in the top-left corner of the Test Explorer sidebar.
   ![Suite Management](/suite/Test-IDE.png)
   
2. **Name the Suite:** Provide a clear, descriptive name in the "Suite Name" field (e.g., "Authentication Flow" or "Checkout Cart").
   ![Suite Creation Dialog](/suite/Suite-Creation-Dialog.png)

3. **Select the Suite Type:**
   Robonito enforces architecture separation natively. Choose the platform target for this suite:
   - **Web:** For browser-based UI functionality tests.
   - **API:** For executing and validating backend endpoint tests.
   - **Mobile:** For running native iOS and Android application UI tests.
   - *(Note: Advanced agent and desktop capabilities are dynamically supported depending on your Robonito environment version).*

4. **Finalize:** Click **Create** to officially save the suite to your workspace workspace.

---

## 2. Managing Your Suites

You can individually manage each suite directly from the sidebar. 
Click the **three vertical dots (ellipsis)** next to any suite name to open the suite actions menu:

![Suite Action](/suite/SuiteActions.png)

From this menu, you have comprehensive control over the suite's lifecycle:
- **Add Test Case:** Immediately scaffold a blank test case perfectly matching the suite's architecture type.
- **Schedule:** [Automate the continuous execution](../schedules) of all tests within this suite at specific recurring intervals natively within the cloud.
- **Report:** Instantly view high-level [Execution Reports](../reports) mapping test pass/fail ratios for this specific feature module.
- **Generate Code:** Quickly export underlying code snippets generated by the test steps inside this suite.
- **Delete Suite:** Safely and permanently remove the entire suite and all its child test cases.

---

## 3. Global Suite Execution

One of the largest benefits of grouping tests into suites is the ability to trigger **batch executions**. 

Instead of painstakingly running 50 test cases manually one by one, you can execute the entire suite at the click of a button:

* **[Local Execution](../local):** Leverage your host machine's resources to execute the entire suite of test cases locally in real-time.
* **[Cloud Execution](../cloud):** Offload the heavy execution processing completely to Robonito's scalable cloud infrastructure.

As the suite runs iteratively, Robonito seamlessly aggregates the outcomes of every grouped test case into unified, easy-to-read completion metrics.
