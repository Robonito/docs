# Install Robonito on macOS

This guide explains exactly how to install and securely set up the **Robonito** desktop application on macOS systems.

---

## ⚠️ Important Note – Intel Macs Not Supported

> **Robonito currently does NOT support Intel-based macOS systems.**  
> The application is built **strictly for Apple Silicon (ARM64)** architectures.

### ✅ Supported Mac Devices (Apple Silicon)
- **MacBook Air:** M1, M2, M3
- **MacBook Pro:** M1, M2, M3
- **Mac Mini:** M1, M2
- **Mac Studio:** M1, M2
- **Mac Pro:** Apple Silicon Models

### ❌ Not Supported
- Any **Intel-based Macs** (e.g., Intel Core i5, i7, i9)
- Any macOS operating system running on an Intel processor.
- **Rosetta 2 emulation** is actively **not supported** at this time.

*(Please carefully verify your Mac’s internal processor via `Apple Menu > About This Mac` before attempting to download the software.)*

---

## System Requirements

Before downloading and installing Robonito, please ensure your Apple device meets the following exact requirements:

- **Operating System:** macOS 11 (Big Sur) or any later version.
- **Architecture:** Apple Silicon (ARM64) **strictly required**.
- **Browser:** Google Chrome (the latest stable version is highly recommended for best automation execution).
- **Internet Connection:** An active broadband connection is strictly required for initial installation and background updates.

## Installation Steps

1. **Download:** Securely download the Robonito macOS installer file (`.dmg`) from the designated link above.
2. **Mount:** Double-click to open and mount the downloaded `.dmg` image file on your desktop.
3. **Install:** Drag and drop the **Robonito** application icon directly into the adjacent **Applications** directory folder on the prompt.
4. **Launch:** Safely eject the mounted disk image and finally open the Robonito app directly from your macOS Applications folder.

---

## Initial System Permissions

On its very first launch, macOS core security features will safely request the following native privacy permissions:

- **Accessibility:** Required for the testing engine to actively interact with UI elements on your screen.
- **Screen Recording:** Strictly required to capture live screenshot visuals, assertion evaluations, and element targeting during your test recordings.

You **must successfully grant** both of these local privacy permissions in your Mac's **System Settings** to fully enable Robonito's recording and web automation features.
