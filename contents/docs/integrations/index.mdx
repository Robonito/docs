## Understanding Robonito Integrations

Integrations allow you to seamlessly connect Robonito with the other core enterprise tools your team uses daily, streamlining your automation workflows and keeping all testing activities deeply connected to your existing ecosystem.

> **Important:** Integrations are secure and specifically scoped to each **workspace**. This means every individual Robonito workspace will securely maintain its own entirely separate, isolated set of integrations.

---

## Accessing Integrations

To manage or add new external connections, simply click on the **Integrations** tab located in the left sidebar of your main Robonito dashboard.

## Supported Integrations

Robonito currently supports native plug-and-play integrations with several popular enterprise tools:

### [Jira](./jira)
Natively connect Robonito to your Atlassian Jira projects to:
- Directly create and link new Jira issues straight from your failed test execution reports.
- Automatically log bugs, tasks, or stories without leaving the Robonito dashboard.

### [Email (IMAP)](./email)
Give Robonito secure IMAP access to test email inboxes to:
- Automatically read incoming **One-Time Passwords (OTPs)** or verification links directly from targeted test emails.
- Perfectly automate secure test cases that strictly require two-factor email verification steps.

### [GitHub](./github)
Connect Robonito directly to your GitHub source code repositories to:
- Continually trigger automated Robonito regression tests based on live VCS pipeline events.
- **Pull Request:** Automatically run your test suites when a new PR is opened or updated.
- **Push:** Execute structural tests immediately after new code is pushed to a specified branch.
- **Merge:** Initiate final regression tests when changes are formally merged into your main production branches.

### [Authenticator Apps](./authenticator-apps)
Securely bind Robonito to standard TOTP protocols (like Google Authenticator, Microsoft Authenticator, or Authy) to:
- Automatically generate and securely inject dynamic **One-Time Passwords (OTPs)** directly into active test case nodes.
- Flawlessly unblock automated tests that strictly require app-based standard 2FA verification layers.

*These powerful integrations directly help you bridge Robonito seamlessly with your existing development, security, and testing ecosystems—making your automation journey far more efficient and heavily collaborative.*
