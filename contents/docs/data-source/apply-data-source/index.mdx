Once you have successfully uploaded a **Data Source** (like a `.csv` or Excel file), the next step is to **connect it to your test case**. This process allows you to perform data-driven testing by dynamically filling your test inputs with the rows from your attached data file.

---

## Steps to Apply a Data Source

To begin attaching data values, follow these initial steps:

### 1. Open the Test Explorer

1. Navigate to the **Test Explorer** from the main Robonito dashboard.
2. Select the specific test case you want to parameterize with your data source.

![Test Explorer View](/data-source/test-explorer.png)

### 2. Select an Input Node

1. Click on any test step within the timeline where data needs to be populated (e.g., a **Fill Input** node).
2. The step configuration panel will open on the right, allowing you to edit the **Fill Input Value**.

![Edit Test Node Panel](/data-source/edit-node.png)

---

## How to Insert a Data Source Value

Within the **Fill Input Value** field, you can fetch your data by following these exact steps:

1. **Trigger the Menu:** Start typing `[[` in the input field. This automatically opens a popup displaying all your uploaded data sources.
2. **Select the File:** Click on the desired **file name** from the dropdown list.
3. **Select the Column:** A secondary popup will immediately appear. Select your desired **column name** from that specific data source.
4. **Completion:** After selecting the column, the field expression will automatically complete using the following syntax:

   ```text
   [[filename.columnName]]
   ```

### Working Example

If your uploaded file is officially named `users.csv` and it contains a column named `email`, your field value would look like this:

```text
[[users.email]]
```

![Select Data Source Popup](/data-source/fill-data-source.png)  
![Select Data Source Column Popup](/data-source/fill-data-source-column.png)

This explicit syntax tells Robonito to dynamically pull the value from the `email` column for each corresponding row during the automated execution of the test case.

---

## Important Things to Remember

Keep the following rules in mind when working with data sources:
- **Trigger Key:** The data source popup **only** appears when you explicitly start typing exactly `[[`.
- **Exact Matches:** The file name and column name must match exactly with what was originally uploaded.
- **Verification:** You can safely preview and verify your data bindings before running the actual test case.
