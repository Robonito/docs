# Execution Reports

The Execution Report provides a detailed history of your API test case runs, allowing you to monitor test results, analyze API performance, and debug any failures. Each report contains comprehensive information about the requests made, the responses received, and the outcomes of all assertions.

## Accessing the Execution History

To view the execution reports for your API tests:

1. **Navigate to Logs:** Click the **Logs** button located in the top navigation bar of the Robonito interface.  
   *(Suggestion: Add a screenshot highlighting the Logs button)*

2. **View the History Table:** A side panel or drawer will open, displaying a chronological table of your past test executions. Each row represents a single run of an API test case or test suite.  
   *(Suggestion: Add a screenshot of the execution history table)*

3. **Open a Specific Report:** Click on any row within the table to view the detailed execution report for a specific test run.

## Understanding the Report Details

Clicking on a specific execution opens a detailed view, which is typically organized into the following sections:

### Request Details

This section shows the exact request parameters that were sent during the API test run:

- **HTTP Method:** The target method used (e.g., `GET`, `POST`, `PUT`, `DELETE`).
- **Endpoint URL:** The full targeted URL.
- **Headers:** A list of all HTTP headers sent with your request.
- **Request Body:** The payload sent with the request (if applicable, e.g., for `POST` or `PUT` requests).
- **Configuration:** Any specific configuration parameters that were applied during this run (e.g., selected environment variables).

### Response Details

This section displays the full response received from the API server:

- **Status Code:** The HTTP status code returned (e.g., `200 OK`, `404 Not Found`, `500 Internal Server Error`).
- **Response Time:** The duration (in milliseconds) it took to receive the response after sending the request.
- **Headers:** A list of all HTTP headers received from the server response.
- **Response Body:** The data payload returned by the server.

### Assertion Results

If your API test case includes configured assertions, this section shows their outcomes:

- **Assertion List:** Displays each assertion configured for the request.
- **Status:** Indicates whether each assertion **Passed** (`✅`) or **Failed** (`❌`).
- **Comparison:** For failed assertions, it shows the **Expected Value** compared directly against the **Actual Value** received to help you debug the issue.
