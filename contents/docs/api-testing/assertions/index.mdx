# Assertion Types in Robonito API Testing

Assertions in Robonito allow you to validate API responses to ensure correctness, reliability, and performance.

They help verify:

- HTTP Status Codes  
- Response Time  
- Response Body (JSON / Text)  
- Response Headers  
- Response Size  
- Data Structure  

Assertions are essential for validating API behavior during automated test execution.


## How Assertions Work in Robonito

When an API request is executed:

1. Robonito sends the request.
2. The server returns a response.
3. Configured assertions are evaluated against the response.
4. The test passes or fails based on assertion results.

If any assertion fails, the test case is marked as **Failed**.


## Supported Assertion Types


### 1. Status Code

#### Purpose
Validates the HTTP status code returned by the API.

#### When to Use
- Validate successful responses (`200`)
- Validate resource creation (`201`)
- Validate client errors (`400`, `404`)
- Validate authorization errors (`401`, `403`)
- Validate server errors (`500`)

#### Configuration Example

**Assertion Type:** `Status Code`  
**Operator:** `=`  
**Expected Value:** `200`

**Result:** The response status must be exactly 200 (OK).


### 2. Response Time (ms)

#### Purpose
Validates the API response time in milliseconds.

#### When to Use
- Performance validation
- SLA monitoring
- Load validation scenarios

#### Configuration Example

**Assertion Type:** `Response Time (ms)`  
**Operator:** `<`  
**Expected Value:** `500`

**Result:** The API must respond in less than 500 milliseconds.


### 3. Response Body (JSON)

#### Purpose
Validates specific values inside the JSON response body using JSONPath.

#### When to Use
- Verify specific fields such as `username`, `email`
- Validate transaction status
- Check nested data structures
- Validate arrays


#### Sample Response

```json
{
  "user": {
    "id": 101,
    "role": "admin"
  },
  "status": "success"
}
```


#### Example 1 – Validate User Role

**Assertion Type:** `Response Body (JSON)`  
**JSONPath:** `$.user.role`  
**Operator:** `=`  
**Expected Value:** `admin`

Result: Verifies that the user role is `admin`.


#### Example 2 – Validate Status

**JSONPath:** `$.status`  
**Operator:** `=`  
**Expected Value:** `success`

Result: Verifies that the API returned a successful response.


#### Example 3 – Validate Array Length

**JSONPath:** `$.users.length()`  
**Operator:** `>`  
**Expected Value:** `0`

Result: Verifies that at least one user exists.


### 4. Response Body (Text)

#### Purpose
Validates plain text responses.

#### When to Use
- HTML responses
- Error messages
- Raw text APIs

#### Example

**Assertion Type:** `Response Body (Text)`  
**Operator:** `contains`  
**Expected Value:** `Welcome`

Result: Verifies that the response contains the word `Welcome`.


### 5. Response Header

#### Purpose
Validates HTTP response headers.

#### When to Use
- Content-Type validation
- Cache validation
- Security header validation


#### Example – Validate Content Type

**Assertion Type:** `Response Header`  
**Header Name:** `Content-Type`  
**Operator:** `contains`  
**Expected Value:** `application/json`

Result: Verifies the API returns a JSON response.


### 6. Response Size (Bytes)

#### Purpose
Validates the size of the response payload.

#### When to Use
- Prevent oversized payloads
- Performance optimization validation

#### Example

**Assertion Type:** `Response Size`  
**Operator:** `<`  
**Expected Value:** `2048`

Result: Verifies the response size is under 2048 bytes.


## Supported Operators

Robonito supports the following operators:

- `=` — Equal to
- `!=` — Not equal to
- `>` — Greater than
- `<` — Less than
- `>=` — Greater than or equal
- `<=` — Less than or equal
- `contains` — Value contains substring
- `not contains` — Value does not contain substring
- `is empty` — Field is empty
- `is not empty` — Field is not empty
- `matches regex` — Matches regular expression


## How to Add an Assertion in Robonito

Follow these steps:

### Step 1: Open API Test Case

Navigate to your API test case in Robonito.

### Step 2: Open Assertions Section

Click on the **Assertions** tab.

### Step 3: Add Assertion

Click **Add Assertion**.

### Step 4: Configure Assertion

- Select Assertion Type
- Choose Operator
- Enter Expected Value
- If using JSON validation, provide the JSONPath

### Step 5: Save and Run

Save the test case and execute it.


## Complete Example – API Test Scenario

### API Request

```bash
GET /api/users/101
```

### Expected Assertions

- Status Code | = | 200
- Response Time | < | 500
- Response Body (JSON) | $.user.id | = | 101
- Response Body (JSON) | $.user.role | = | admin
- Response Header | Content-Type | contains | application/json

If all validations pass, the test is marked as **Passed**.

If any validation fails, the test is marked as **Failed**.
