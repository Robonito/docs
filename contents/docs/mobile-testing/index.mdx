# Mobile Testing with Robonito

Robonito provides a powerful, highly intuitive architecture to extensively test mobile applications natively, ensuring they function correctly and smoothly across vastly different operating systems, screen sizes, and virtual devices. This section comprehensively guides you through the available tools, configurations, and advanced features distinctly designed for recording, orchestrating, and analyzing mobile test cases using local emulators.

---

## Supported Platforms

Robonito's native automation engine effectively supports end-to-end mobile testing for the following ecosystems:
- 🤖 **Android Applications** (`.apk`)
- 🍏 **iOS Applications** (`.ipa`)
- 💻 **Emulator-Based Execution** (Android Studio / Appium)

---

Navigate through the core tutorial topics below to safely configure your local setup and begin organically recording robust mobile tests in Robonito:

- ⚙️ **[Configure Mobile Setup](./mobile-setup)**

- 🎥 **[Start Recording a Mobile Test](./recording)**  
  Learn how to natively upload your `.apk` or `.ipa` app files, automatically boot the emulator, and intuitively record your touch interactions as automation steps.

- 🅰️ **[Use Variables in Mobile Tests](./variables-in-mobile)**

- ✅ **[Add Mobile Assertions](./mobile-assertions)**

- 📊 **[Understand Mobile Execution Reports](./mobile-execution-report)**
