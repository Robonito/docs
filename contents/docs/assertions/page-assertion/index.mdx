# Page Assertions

Page Assertions are used to validate page-level properties during test execution. They ensure that your application successfully navigates to the correct page and that the page loads exactly as expected.

The **Add Page Assertion** dialog appears when you choose to add a page-level validation during an active recording or while editing an existing test step.

---

## Page Assertion Types

Robonito supports the following types of page-level assertions to validate your application's routing and metadata:

### 1. Page Title
Verifies the browser tab's title for the currently loaded page.

**Configuration Options:**
- **Expected Title (\*)** – Enter the exact or partial string of the expected page title.
- **Match Type** – Choose how the title should be evaluated:
  - **Exact Match:** The title must match the input string entirely.
  - **Partial Match:** The title must simply contain the input string.
- **Case Sensitive** – Enable this option to strictly enforce capitalized and lowercase matching.

**Use Case:**  
Ensure the user is navigated to the correct page after completing an action (e.g., verifying the title says "Dashboard" after a successful login).

---

### 2. Page URL
Validates the explicit web address (URL) of the currently loaded page.

**Configuration Options:**
- **URL (\*)** – Enter the expected page URL string.

**Use Case:**  
Confirm correct navigation or routing flow after specific page transitions (e.g., verifying the URL changes from `/cart` to `/checkout`).

---

## Adding a Page Assertion

To finalize and add a page assertion to your test:

1. **Review:** Double-check your expected Title or URL configuration.
2. **Add:** Click the **Add Assertion** button to attach the page assertion to the selected test step.

During execution, Robonito evaluates the page assertion precisely at that step and marks it as either **Passed** or **Failed** in your test's execution report.

---

## Benefits of Page Assertions

Routinely using Page Assertions in your test suites helps ensure:
- Correct navigation and routing workflows.
- Accurate page rendering and metadata generation.
- Reliable validation of complex user journey transitions.
