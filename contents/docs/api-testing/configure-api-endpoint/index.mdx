# Configuring an API Endpoint

Before you can execute an API test in Robonito, you must configure its endpoint details. This process includes specifying the target URL, any required parameters, HTTP headers, the request body, and your authentication credentials.

Follow these steps to configure your API endpoint successfully:

## 1. Endpoint URL
The base address where your API request will be sent.
- **Field:** `Endpoint URL`
- **Action:** Enter the complete URL of the target API endpoint.
- **Example:** `https://api.example.com/v1/users`

---

## 2. Query Parameters
Query parameters are used to modify the request and are typically appended to the end of the URL. 

You can add query parameters in two ways:
- **Directly in the URL:** Append them directly to the Endpoint URL using standard syntax (e.g., `?key=value&key2=value2`).
- **Using the Parameters Tab:**
  1. Navigate to the **Parameters** tab.
  2. Click the **Add new +** button.
  3. Enter the **Key** and **Value** for each parameter. Robonito will automatically append these to the URL when the request is sent.

---

## 3. Headers
HTTP headers provide the server with additional information about the request or the client.

- **Action:**
  1. Navigate to the **Headers** tab.
  2. Click the **Add new +** button for each header you want to add.
  3. Enter the **Header Name** (e.g., `Content-Type`) and its corresponding **Value** (e.g., `application/json`).
- **Common Headers:** `Content-Type`, `Authorization`, `Accept`, `User-Agent`, and custom headers like `X-API-Key`.

---

## 4. Request Body
If your API request requires you to send data (typically for `POST`, `PUT`, and `PATCH` methods), you will configure it here.

- **Format:** Robonito currently supports the **JSON** format for the request body.
- **Action:** Enter your request payload into the body section, ensuring it uses valid JSON syntax.
- **Example (Creating a User):**
  ```json
  {
    "username": "testuser",
    "email": "test@example.com"
  }
  ```

---

## 5. Authentication
Configure how your API test case will securely authenticate with the target server.

- **Action:** Select your required authentication method from the available options and provide the necessary credentials.
- **Supported Methods:**
  - **Bearer Token:** Provide your token value.
  - **Basic Auth:** Provide the required username and password.
  - **API Key:** Specify the key name, the key value, and its placement (e.g., in the Header or Query Params).
  - *(Additional supported methods will appear here based on your workspace setup)*

---

## Saving Your Configuration

> **Important:** Robonito does **not** automatically save changes as you type in the configuration screen. You must explicitly click the **Save** button (or the equivalent UI action) before navigating away to ensure your endpoint configuration is preserved.

---

## Running Your Test

Once your API endpoint is fully configured and saved, you are ready to execute the request:

1. Ensure you are viewing the fully configured API test case.
2. Click the **Run Test** button (or equivalent), typically located at the top of the screen.
3. Robonito will send the API request, then capture and display the response. This includes the HTTP status code, response headers, response body, and the results of any configured assertions.
