# Variables in Robonito API Testing

Variables allow you to capture specific values from an API response and dynamically reuse them in subsequent API requests. This feature is crucial for building dynamic, chained, and real-world test scenarios.

Variables are commonly used for:
- Authentication tokens
- User or Object IDs
- Order IDs
- Transaction references
- Dynamic response data

Using variables enables validation of full, end-to-end API workflows.

---

## Why Variables Are Important

In real-world applications, APIs often return dynamic data, such as:
- Newly created resource IDs
- Session and access tokens
- Unique reference numbers

Instead of hardcoding these temporary values in your test cases, Robonito allows you to extract and reuse them on the fly. This best practice makes your API tests **reusable, maintainable, reliable, and production-ready.**

---

## How Variables Work

When an API request is executed, the process is as follows:
1. The API returns a response.
2. Robonito extracts a specific target value using a given JSONPath.
3. The extracted value is stored as a variable.
4. The variable can then be used in subsequent API requests using the `{{variableName}}` syntax.

---

## Variable Extraction

Variable extraction allows you to capture data from multiple sources within an API response:
- Response Body (JSON)
- Response Headers
- Response Status Code
- Other supported target sources

---

## Configuring a Variable

To configure variable extraction in Robonito, follow these steps:

![Configuring a Variable](/test-case/variable-setup.png)

1. **Open the Request:** Navigate to your desired API request.
2. **Go to the Variables Tab:** Click on the **Variables** section.
3. **Add an Extraction:** Click the **Extract Variable** button.
4. **Configure Fields:** Fill in the extraction details (explained below).

![Variable Extractions Config](/test-case/variable-extractions.png)

### Variable Name
The custom name you will use to reference the variable later.
- **Examples:** `userId`, `authToken`, `orderId`
- **Usage Syntax:** `{{userId}}`

### Source
Select where the value should be extracted from.
- **Common Source:** `Response Body (JSON)`

### JSONPath
Provide the specific [JSONPath](https://jsonpath.com/) expression to extract the value.

**Example Response:**
```json
{
  "user": {
    "id": 101,
    "email": "test@example.com"
  },
  "token": "abc123xyz"
}
```

**Example JSONPath Expressions:**
- `$.user.id`
- `$.user.email`
- `$.token`

---

## Using Variables in Requests

You can use an extracted variable in subsequent API requests across various fields. Robonito will automatically replace the `{{variableName}}` placeholder with the extracted value at runtime.

### Allowed Locations
Variables can be placed directly within test configurations, including:
- Request URLs
- Query Parameters
- Request Body (JSON)
- HTTP Headers
- Authorization fields

**Example (Request Body):**
```json
{
  "userId": "{{userId}}",
  "status": "active"
}
```

---

## Practical Examples

### Example 1: Extracting a User ID

**Original API Request:**
```http
POST /api/users
```

**Sample Response:**
```json
{
  "id": 101,
  "name": "Arshad"
}
```

**Variable Configuration:**
- **Variable Name:** `userId`
- **Source:** `Response Body (JSON)`
- **JSONPath:** `$.id`

After execution, the runtime stores: `{{userId}} = 101`.

**Using the Variable (Next Request):**
```http
GET /api/users/{{userId}}
```
*At runtime, this is dynamically resolved to: `GET /api/users/101`*

### Example 2: Extracting an Auth Token

**Login API Request:**
```http
POST /api/login
```

**Sample Response:**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**Variable Configuration:**
- **Variable Name:** `authToken`
- **Source:** `Response Body (JSON)`
- **JSONPath:** `$.accessToken`

**Using the Token in Headers:**
```http
Authorization: Bearer {{authToken}}
```
*This allows for seamless authenticated API chaining.*

---

## Common JSONPath Examples

When extracting values from complex JSON structures, use these JSONPath tricks:

- **Extract first item in an array:**  
  `$.users[0].id`
- **Extract a nested value:**  
  `$.data.user.profile.email`
- **Extract the array length:**  
  `$.users.length()`

---

## Variable Scope

Variables extracted during a test execution are scoped locally and are available:
- Within the same test case sequence.
- In subsequent API calls inside the same workflow.
- Only retrieved and resolved during runtime.

---

## End-to-End Workflow Example

Here is how variables enable a complete end-to-end user scenario:

1. **Step 1: Create User**  
   Extract `userId` from the creation response.
2. **Step 2: Login User**  
   Extract `authToken` from the login response.
3. **Step 3: Get User Details**  
   Use variables to request the valid profile:
   ```http
   GET /api/users/{{userId}}
   Authorization: Bearer {{authToken}}
   ```
   *This single workflow validates the full path from creation to authenticated retrieval.*
