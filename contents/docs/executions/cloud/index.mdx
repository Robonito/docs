# Cloud Execution

Cloud Execution allows you to run your automated test cases directly on Robonito’s dedicated cloud servers rather than relying on your local desktop machine. This feature is designed for enterprise scalability, CI/CD integration, and seamless team-based test workflows.

> **Note:** Cloud Execution is currently available as a premium feature.

---

## Key Benefits of Cloud Execution

- **Zero Local Dependency:** Tests are executed entirely in the cloud, unblocking your local system's CPU and memory resources for active development work.
- **Consistent Testing:** Provides a highly stable, controlled environment across all test runs, eliminating the "it works on my machine" discrepancy.
- **Remote CI/CD:** Perfect for integrating into automated deployment pipelines or scheduling continuous overnight test suites without needing a physical desktop turned on.

---

## Enabling Cloud Execution

If Cloud Execution is **not** actively enabled for your workspace:
- The execution action for cloud runs will be visibly locked in the user interface.
- You can request premium access by contacting Robonito Support through Email or our community Slack.

Once enabled by an administrator, the **Cloud** target officially becomes selectable in the standard Execution Configuration menu.

---

## Behavior & Reporting

During an active cloud execution:
- Test steps execute seamlessly in real-time on secure cloud clusters.
- All defined assertions and conditional logic blocks are evaluated.
- Live execution progress is broadcasted back to your local Robonito dashboard.

After execution, a comprehensive **Cloud Execution Report** is immediately generated for your entire team. This report logs historical pass/fail trends, centralizes debugging logs, and helps identify flaky automation test steps across your organization.
