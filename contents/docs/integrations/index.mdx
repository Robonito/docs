# External Integrations

Integrations allow you to seamlessly connect Robonito with third-party bug trackers, version control systems, and two-factor authentication solutions to streamline your entire testing lifecycle.

> **Important:** Integrations are bound strictly to your **active Workspace**. Different workspaces can maintain entirely isolated sets of connected integrations without overlapping credentials.

---

## Accessing Integrations

To configure new external connections:
1. Navigate to the main dashboard.
2. Click on the **Integrations** tab located in the left navigation sidebar.

## Supported Ecosystems

Robonito currently features native plug-and-play combinations with the following enterprise systems:

### [Jira](./integrations/jira)
Connect Robonito specifically to Atlassian Jira to dynamically generate issue tickets. You can configure automation rules to instantly create and link new Jira bugs specifically when cloud execution reports fail.

### [Email Data Sources](./integrations/email)
Connect Robonito to IMAP email inboxes. This complex integration allows your automation engine to fetch real-time One-Time Passwords (OTPs) or click registration links directly out of emails to bypass secure login flows during UI tests.

### [GitHub Actions](./integrations/github)
Bind your repository to trigger cloud testing runs based on live version control events. Run your regression suites directly when a Pull Request is opened, code is pushed, or a branch is merged into production.

### [Authenticator Apps (TOTP)](./integrations/authenticator-apps)
Register a shared Time-Based One-Time Password (TOTP) seed with Robonito—acting just like Google Authenticator or Authy. The engine will programmatically generate and inject valid 2FA codes into your tests during active execution.
