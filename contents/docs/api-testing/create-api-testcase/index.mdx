# Creating an API Test Case

This guide explains how to create a new API test case using Robonito.

## Adding Test Cases to a Suite

In Robonito, test cases are systematically organized within **Test Suites** based on their functionality or modules. To create API tests, ensure you are working within a suite designed for API testing.

### Steps to Create a Test Case

Follow the steps below to create a new API test case:

1. **Select the Target Suite**  
   Choose an existing test suite from the **Target Suite** dropdown. Alternatively, you can type a new name to dynamically create a brand-new suite where this test case will be added.  
   ![Target Suite Selection](/test-case/target-suite.png)

2. **Enter the Test Case Name**  
   Provide a clear and descriptive name in the **Test case name** field to easily identify the test scenario later.  
   ![Test Case Name Field](/test-case/test-case-name.png)

3. **Select the Test Type**  
   Choose **API** as the test case type. This ensures the system recognizes the test as an API automation test and provides the correct configuration options.  
   ![API Test Type Selection](/test-case/api-test.png)

4. **Add a Description (Optional)**  
   Enter a short description explaining the purpose and scope of the test case. Providing a description is highly recommended for team collaboration.

5. **Create the Test Case**  
   Click the **Create** button to save the test case. Once created, your new API test case becomes available for configuring endpoints, adding assertions, and execution.
