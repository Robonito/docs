## Explore the Key Features
### [Workspaces Overview](http://localhost:3000/docs/workspaces)

Workspaces serve as the main organizational units in Robonito, facilitating project organization and team collaboration.

### [Manage Team](http://localhost:3000/docs/teams) 
The team member feature in the Robonito app is essential for workspace management. 
Invite team members via their email addresses for immediate access through their Robonito accounts, enhancing collaboration and boosting productivity.

### [Test Suites](http://localhost:3000/docs/suites)

Test suites group related test cases within a workspace, streamlining your testing efforts.

### [Creating and Managing Test Cases](http://localhost:3000/docs/test-case)  
Creating Test cases in a Robonito is dynamic feature for effective testing. 
where you can confidently add new test cases or edit existing ones to align with current requirements.
Robonito have record icon to capture test steps in real time, ensuring comprehensive and precise documentation.
This assertive approach to test case management enhances clarity and drives efficiency throughout the testing process.

#### [Recording Tests](http://localhost:3000/docs/web-testing/recordtest-case) 
- Click the record icon for your test case.  
- Set initial configurations, such as the target URL and viewport dimensions.  
- Click "Start Recording" to capture your actions in the IDE pane.

### [Executions](http://localhost:3000/docs/executions)

Robonito offers versatile options for executing your tests.

#### [Local Execution](http://localhost:3000/docs/executions/local)  
- Click the play icon on your test case.  
- Select "Local Execution."  
- Watch the browser launch and monitor real-time execution.

#### [Cloud Execution](http://localhost:3000/docs/executions/cloud) 
- Follow a similar process as local execution.  
- Choose "Cloud Execution" for enhanced scaling.

### Advanced Features

#### [Schedule](http://localhost:3000/docs/schedules)  
  Scheduling in Robonito is used for effective test management.
  Users can easily access suite options through the vertical dots menu and select the scheduling option to configure key parameters. 
  This allows them to set the frequency of test runs—daily, weekly, or event-triggered—and align execution timing with development cycles.  
  By utilizing these features, teams can enhance their testing workflow and boost productivity in their quality assurance efforts.

#### [Reports](http://localhost:3000/docs/reports)
The reports is an important feature in a Robonito application which provides essential insights through easy access via the reports icon and suite-specific reports from the vertical dots menu. Users can filter data by suite, status, and executor for targeted analysis. Interactive bar charts visualize execution outcomes, while detailed histories and custom filtering options allow for tailored reporting, significantly boosting testing efficiency.
