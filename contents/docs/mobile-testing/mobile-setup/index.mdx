# Mobile Setup

If your required mobile testing environment is not fully configured, Robonito will display the active warning: **“Mobile Setup Not Complete”**. This warning strictly indicates that the local mobile testing architecture (such as SDKs or Emulators) is currently missing required core components to function.

To continue with your mobile test recording, click **Configure Environment** to begin and complete your mobile setup installation. Once the setup is securely finished, you can freely start recording automated mobile test cases.

![Mobile Setup Configuration Dashboard](/test-case/mobile-setup.png)

---

## Environment Setup Overview

The **Mobile Environment** setup wizard effectively allows you to download and correctly configure:
- Android SDK Toolchains
- Node.js Runtime
- Appium Servers

This setup pipeline instantly prepares your local host system for complex mobile automation testing.

> **Important:** Robonito intelligently manages completely isolated SDKs within its own folders to ensure maximum build stability. This automated setup will **not** affect or overwrite your machine's pre-existing global native environment variables.

---

## Environment Status Dashboard

At the very top of the setup page, you can see a high-level overview of your installation progress:

- **Installation Progress:** A live percentage indicating how much of your environment is physically ready.
- **Platform:** Your actively detected host operating system (e.g., `Windows`, `macOS`).
- **Setup Required:** A visual indicator showing if actionable setup steps are currently required.
- **Install All:** The primary action button used to simultaneously install all missing components automatically.

Simply click **Install All** to safely fetch and install all missing dependencies.

---

## Required Environment Components

The following native software components must be successfully installed on your machine for mobile testing to function. 

*(Note: Robonito will display a red `MISSING` status next to any tool that has not been securely configured yet.)*

| Component | Description | Default Status |
| :--- | :--- | :--- |
| **Java Development Kit (JDK)** | The baseline Java runtime strictly required for compiling Android build tools. | `MISSING` |
| **Android Command Tools** | The core Android SDK Manager & AVD (Android Virtual Device) Manager utilities. | `MISSING` |
| **Platform Tools (ADB)** | Provides the Android Debug Bridge pipeline to communicate efficiently with virtual devices. | `MISSING` |
| **Android Emulator** | An automated Virtual Device (e.g., a virtual Pixel 9 smartphone) exclusively used for test rendering. | `MISSING` |
| **Node.js Runtime** | An isolated, local Node.js JavaScript engine explicitly used for operating the Appium server. | `MISSING` |
| **Appium Server** | The primary automation server engine used to translate your web tests (v2.12 or newer). | `MISSING` |
| **UiAutomator2 Driver** | Appium's core Android automation and UI manipulation accessibility driver. | `MISSING` |

---

## How to Install Components

Follow these exact steps to automatically fetch all the listed structural requirements above:

1. **Launch:** Open the main Robonito application interface.
2. **Navigate:** Proceed directly to the **Mobile Setup** dashboard locally.
3. **Execute:** Click the **Install All** button natively located at the top of the interface.

Robonito will safely and autonomously sequence, download, and install all missing architectural components in strict isolation for you.
