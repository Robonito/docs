# Test Case Scheduling

Robonito allows you to schedule test cases to run automatically at specific times and recurring intervals. Scheduling is essential for continuous validation methodologies—such as regression testing, nightly parallel runs, and unattended pipeline integrity checks.

---

## Creating a New Schedule

To create an autonomous schedule, navigate to the **Schedules** section in the left sidebar and click the **New Schedule** button. 

![Schedules Dashboard Overview](/test-case/schedule.png)

### 1. Configure the Timeline

The configuration wizard requires the following parameters:

![New Schedule Configuration](/test-case/new-schedule.png)

- **Schedule Name:** Enter a uniquely identifiable name (e.g., *Nightly API Verification*). The maximum length is 50 characters.
- **Frequency:** Define how often the scheduled suite should execute (e.g., *Daily*, *Weekly*).
- **Timezone:** Select the geographical timezone applicable to your automated execution. Your trigger times are strictly anchored to this timezone.
- **Execution Time:** Specify the precise 24-hour time at which execution should trigger.
- **Environment:** Select the targeted execution environment (e.g., `LOCAL` or `CLOUD`).

### 2. Select Test Cases

Directly beneath the timeline configuration, you must select the test cases that belong to this schedule instance:
- Use the search bar to filter test cases by name or tag.
- Select checkboxes securely to add test cases to the queue.
- Tests are executed sequentially exactly as grouped.

*Note: You cannot save a schedule unless at least one test case has been actively selected.*

### 3. Save the Schedule
Once configured, click **Create Schedule**.
The automation engine will immediately queue your test suites to recursively execute at the designated time.

## Managing Your Schedules

All saved schedules are permanently maintained in the native **Schedules** dashboard.

![Schedules Table Dashboard](/test-case/schedule-report.png)

### Schedule Table Overview

| Column | Description |
| :--- | :--- |
| **Name** | Your uniquely assigned schedule identifier. |
| **Status** | The timeline status (`ACTIVE` means the schedule is formally enabled and counting down). |
| **Upcoming Run** | A dynamic countdown to the exact next scheduled execution cycle. |
| **Frequency** | The recurring cadence and timezone specification. |
| **Scheduled By** | The workspace user who originally committed the scheduled loop. |
| **Created** | The creation timestamp. |
| **Actions** | Use the actions menu to **Pause**, **Edit**, or **Delete** the active schedule permanently. |

---

## Scheduled Execution Behavior

When a scheduled automated workflow successfully triggers:
1. All selected test cases execute autonomously.
2. The automation engine runs directly inside the specific environment configured (Local or Cloud).
3. Test progression is recorded sequentially without manual interaction.
4. Comprehensive test execution reports are automatically generated for team review upon completion.
