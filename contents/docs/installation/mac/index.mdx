# Install Robonito on macOS

This guide explains how to install the **Robonito** desktop application natively on macOS systems.

---

## ⚠️ Important Note: Apple Silicon Only

**Robonito is built strictly for Apple Silicon (ARM64) architectures and does NOT support Intel-based Macs.**

- **Supported:** M1, M2, M3, M4 chips (MacBook Air/Pro, Mac Mini, Mac Studio, Mac Pro).
- **Unsupported:** Intel Core processors. Rosetta 2 emulation is also completely unsupported.

Check your hardware via `Apple Menu > About This Mac` before downloading the software.

---

## 1. Download the Installer

Download the official Robonito disk image file specifically compiled for macOS ARM64 architectures:

📥 **[Download Robonito for macOS (v2.5.0)](https://robonito-prod-builds.s3.ap-south-1.amazonaws.com/Robonito-2.5.0-arm64.dmg)**

---

## 2. Install the Application

1. Double-click to mount the downloaded `.dmg` file.
2. In the visual prompt window, drag the **Robonito** application icon directly into your **Applications** folder shortcut.
3. Keep the file transfer window open until the copy completes, then eject the `.dmg` installer.
4. Launch Robonito directly from your macOS Applications directory or Launchpad.

---

## 3. Grant Privacy Permissions

During your first test recording session, macOS will actively block the engine and request standard privacy permissions. You must navigate to your Mac's **System Settings > Privacy & Security** and manually enable:

1. **Accessibility:** Required for the testing engine to interact securely with UI elements on your screen.
2. **Screen Recording:** Required to capture failure frame screenshots and evaluate visual assertions during test automation workflows.
