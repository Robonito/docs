Robonito makes it incredibly easy to upload and manage dynamic test data for your automated test cases using standard **CSV** or **Excel** files. This is the very first step to unlocking the power of **Data-Driven Testing**.

---

## Steps to Upload a Data Source

Follow these steps to successfully upload your data files:

### 1. Open the Data Source Page

- Navigate to the **Dashboard** from the main Robonito interface.
- In the left **sidebar**, click on the **Data Source** icon to open the data management page.

![Data Source Page](/data-source/data-source.png)

### 2. Upload Your File

- Click on the **Browse File** button.
- Choose a file in either **`.csv`** or **Excel (`.xlsx`)** format from your local computer.
- Once selected, the file will automatically upload and begin processing.

![File Upload Dialog](/data-source/file-dialog.png)

### 3. Preview Uploaded Data

- After a successful upload, you will immediately see a **preview of your data** presented in a clean table format.
- The interactive preview table allows you to:
  - **Scroll** through all populated rows and columns.
  - **Search** for specific values using the fast and responsive search box.
  - Verify your data's structure and integrity before applying it to any automated tests.

![Data Preview Table](/data-source/data-preview.png)

---

## Supported File Types

Robonito currently supports the following standard file formats:

- **`.csv`** – Comma-Separated Values
- **`.xlsx`** – Excel Spreadsheets

Each row in your uploaded file represents a distinct data entry that can be iteratively used during your test case execution.

---

## Best Practices

To ensure flawless data-driven testing execution, follow these guidelines:

- **Header Columns:** Ensure your data file uses top-row **header columns** (e.g., `email`, `password`) that exactly match the placeholders used in your recorded test cases.
- **Clean Formatting:** Completely avoid using special characters, blank headers, or merged cells in Excel for the highest compatibility.
- **Always Preview:** Always use the built-in preview table to visually verify your data's integrity before running your final test suite.
